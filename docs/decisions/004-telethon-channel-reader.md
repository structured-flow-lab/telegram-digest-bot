# ADR 004 — Telethon для чтения публичных каналов

## Статус
Принято

## Контекст

Нужно читать посты из публичных Telegram-каналов программно.
У Telegram есть два пути для этого:

1. **Bot API** (`python-telegram-bot`) — официальный, но боты не могут читать
   чужие каналы без добавления бота в качестве администратора.
2. **MTProto (user account)** — официальный протокол, но требует авторизации
   от имени реального аккаунта (не бота). Telethon — Python-библиотека для MTProto.

## Решение

Использовать **Telethon** для чтения каналов от имени пользовательского аккаунта.
Bot API (`python-telegram-bot`) остаётся для интерфейса бота (команды, ответы).

Это два разных клиента в одном приложении:
- `TelegramClient` (Telethon) — читает каналы по `api_id` / `api_hash`
- `Application` (python-telegram-bot) — принимает команды от владельца

Session-файл Telethon хранится в `data/telethon.session`.  
При первом запуске — интерактивная авторизация через терминал (номер телефона + код).

## Последствия

**Плюсы:**
- Telethon — де-факто стандарт для MTProto в Python, активно поддерживается.
- Читает любые публичные каналы без ограничений Bot API.
- Возвращает полные данные поста: текст, views, reactions, ссылка, дата.
- Поддерживает пагинацию и фильтрацию по дате из коробки.

**Минусы:**
- Требует `api_id` и `api_hash` с my.telegram.org (отдельно от бот-токена).
- Session-файл нужно защищать (как и `.env`) — это авторизация пользователя.
- При деплое на Railway нужен persistent volume для `data/`.
- Telegram может ограничить частоту запросов (FloodWait) — нужна обработка.

## Ограничения

- Только **публичные** каналы. Приватные каналы — вне scope Personal MVP.
- Частота запросов: соблюдать паузы между запросами к разным каналам.
- FloodWait: при получении ошибки — ждать указанное время, не ретраить немедленно.

## Альтернативы

- **RSS-парсинг** — не все каналы имеют RSS. Нет views/reactions. Отклонено.
- **Telegram Bot API** — не позволяет читать чужие каналы. Отклонено.
- **telegram.js (Node.js)** — менее стабильный, требует смены стека. Отклонено.
